## 🚨 The Problem
 
Large Language Models (LLMs) like GPT-4o, Claude, Gemini, Llama, and Mistral are being adopted at an unprecedented pace across industries. Millions of people use them daily for research, medical questions, legal advice, financial decisions, and historical fact-checking.
 
**The dangerous truth: LLMs hallucinate.**
 
Hallucination in AI refers to when a model generates content that is factually incorrect, fabricated, outdated, or entirely invented — but presented with full confidence as if it were true. This is not a rare edge case. It is a systematic, measurable, and dangerous problem.
 
### Real-world consequences of LLM hallucinations:
 
| Domain | Risk |
|---|---|
| **Medicine** | A patient asks an LLM about drug dosages or symptoms. The model confidently provides outdated or incorrect information. The patient follows it. This can cause serious harm or death. |
| **Law** | A lawyer uses an LLM to find case citations. The model fabricates non-existent court cases with realistic-sounding names and dates. The lawyer submits them in court. |
| **Finance** | An investor asks about earnings reports or regulations. The model provides incorrect figures, leading to bad financial decisions. |
| **History** | Factual questions about historical events are among the most hallucinated categories. LLMs frequently contradict established historical facts with confident, detailed, but completely wrong responses. |
| **Science** | Research summaries generated by LLMs may cite non-existent papers or misrepresent findings from real ones. |
 
> **History and Technology domains showed the highest hallucination rates in this study — proving that even "factual" domains are not safe from AI fabrication.**

## 💡 Why This Project Matters
 
This project was built to answer a critical question that every organization deploying AI must ask:
 
**"Which AI models can we actually trust, in which domains, and what can we do about it?"**
 
As AI becomes embedded in healthcare systems, legal platforms, educational tools, and financial services — the ability to detect, measure, and mitigate hallucinations is no longer optional. It is a matter of safety, trust, and accountability.
 
### The core insight behind this project:
 
LLMs are extraordinarily powerful tools — but they are not oracles. They are pattern-matching systems trained on vast amounts of text. When asked questions beyond their reliable knowledge, they do not say "I don't know." They generate a plausible-sounding answer. That behavior, when unchecked, is dangerous.
 
This dashboard provides:
- **Quantified evidence** of hallucination rates across models and domains
- **Risk assessment** identifying which models and domains are highest risk
- **Mitigation analysis** showing which strategies actually reduce hallucinations
- **Actionable insights** that organizations can use when deploying LLMs responsibly

## 📊 Key Findings
 
After analyzing **200 LLM responses** spanning **September 2024 – April 2025** across **5 major AI models**:
 
### 🔴 Overall Hallucination Rate: 34.5%
More than 1 in 3 responses contained some form of hallucination. This is alarmingly high for systems being used in high-stakes decision-making.
 
### 📌 Most Common Hallucination Type: Factual Contradiction (24 cases)
The most frequent hallucination was direct contradiction of established facts — where the model stated something confidently that is demonstrably false.
 
### 🌍 Highest Risk Domain: History (70% hallucination rate)
Historical prompts showed the highest hallucination rate. This is particularly concerning because history is often perceived as verifiable — yet LLMs frequently fabricate dates, events, and figures with alarming confidence.
 
### 🏥 Medical Domain: High Risk at 30%
Nearly 1 in 3 medical responses contained hallucinations. Given that people increasingly turn to AI for health guidance, this represents a serious public health concern. **Taking medical advice from an LLM without professional verification can be genuinely dangerous.**
 
### 🤖 Model Performance Comparison:
| Model | Hallucination Rate |
|---|---|
| Mistral-Large | Highest |
| Llama-3.1-70B | High |
| GPT-4o | Medium |
| Gemini-1.5-Pro | Medium |
| Claude-3.5-Sonnet | Lowest |
 
### ✅ Most Effective Mitigation: RAG (Retrieval-Augmented Generation)
RAG — which grounds model responses in verified external documents — was the most frequently used and effective strategy for reducing hallucinations (42 cases).
 
### 🔍 Intrinsic vs Extrinsic:
The majority of hallucinations were **extrinsic** (58 cases) — meaning the model added information that was not in the source material, rather than contradicting it directly.

## 📁 Dataset
 
| Property | Detail |
|---|---|
| **Records** | 200 LLM responses |
| **Time Period** | September 2024 – April 2025 |
| **Models Covered** | GPT-4o, Claude-3.5-Sonnet, Gemini-1.5-Pro, Llama-3.1-70B, Mistral-Large |
| **Domains** | Medicine, Finance, Law, Science, Technology, History, Politics, General |
| **Languages** | English, French, Spanish, Mandarin, Arabic, Hindi, German |
| **Key Columns** | model_name, domain, hallucination_label, hallucination_type, severity, mitigation_strategy, annotation_confidence, intrinsic_or_extrinsic, prompt_type, language |
| **Source** | [Kaggle — LLM Hallucination Benchmark Dataset](https://www.kaggle.com/datasets/alitaqishah/llm-hallucination-benchmark-dataset) by [Ali Taqi Shah](https://www.linkedin.com/in/alitaqishah/) |
 
### Hallucination Types Analyzed:
- **Factual Contradiction** — Directly contradicts known facts
- **Overclaim** — Exaggerates or overstates information
- **Unverifiability** — Makes claims that cannot be verified
- **Incompleteness** — Omits critical information
- **Entity Error** — Confuses names, dates, places
- **Outdatedness** — Provides information that was once true but is no longer accurate
- **Relation Error** — Incorrectly describes relationships between concepts

## 🏗️ Project Architecture
 
```
┌─────────────────────────────────────────────────────────────┐
│                     DATA PIPELINE                           │
│                                                             │
│  Raw Dataset (CSV)                                          │
│       ↓                                                     │
│  Python / Pandas ──── Data Cleaning & Transformation        │
│       ↓                                                     │
│  MySQL (Local) ─────── Relational Database Storage          │
│       ↓                                                     │
│  AWS RDS (MySQL) ───── Cloud Database Migration             │
│       ↓                                                     │
│  AWS S3 ────────────── Raw Data Backup & Storage            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│                   VISUALIZATION LAYER                        │
│                                                             │
│  Power BI Desktop ──── DAX Measures & Interactive Dashboard  │
│       ↓                                                     │
│  Power BI Service ───── Published Public Dashboard          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
 
### Data Flow:
1. **Raw CSV** ingested and inspected
2. **Python/Pandas** cleans nulls, fixes date formats, standardizes columns
3. **MySQL** stores structured relational data locally
4. **AWS RDS** hosts the MySQL database on the cloud
5. **AWS S3** stores the raw and cleaned CSV as backup
6. **Power BI** connects to MySQL, applies DAX measures, renders dashboard
7. **Power BI Service** publishes the dashboard as a public URL

## 🚀 How to Run Locally
 
### Prerequisites
- Python 3.8+
- MySQL 8.0+
- Power BI Desktop (free)
- MySQL Connector/NET
### Step 1 — Clone the repository
```bash
git clone https://github.com/D1Massacre007/llm-hallucination-analytics.git
cd llm-hallucination-analytics
```
 
### Step 2 — Install Python dependencies
```bash
pip install pandas sqlalchemy pymysql
```
 
### Step 3 — Clean the dataset
```bash
python python/clean_data.py
```
 
### Step 4 — Set up MySQL database
```sql
CREATE DATABASE llm_hallucination;
```
 
### Step 5 — Load data into MySQL
```bash
mysql -u root -p llm_hallucination < sql/llm_hallucination.sql
```
 
### Step 6 — Open Power BI Dashboard
1. Open `powerbi/LLM_Hallucination_Dashboard.pbix`
2. Connect to your local MySQL database
3. Refresh data ✅
